Activity associationc back to ToC or Class ToC

IRI: https://w3id.org/rdip/ActivityAssociation

A qualified PROV-O association node linking a ResearchActivity to an agent (Person or Organization) with an explicit RoleInActivity. Use prov:qualifiedAssociation on the activity to point to this node, then prov:agent and prov:hadRole on the node itself. This pattern replaces the simpler rdip:hasActivityRole + rdip:rolePerformedBy chain for full PROV-O compliance.
has super-classes
Association c

Computing environmentc back to ToC or Class ToC

IRI: https://w3id.org/rdip/ComputingEnvironment

The full hardware and software context in which a computational activity was executed. Records OS version, CPU/GPU hardware, RAM, and links to a machine-actionable environment specification (e.g. a Docker image digest, conda environment.yml, or Nix flake). Essential for computational reproducibility.
has super-classes
Entity c, Thing c
is in domain of
CPU cores dp, CUDA version dp, GPU model dp, OS version dp, RAM (GB) dp, hardware specification dp, has environment specification op
is in range of
executed in op

Random seedc back to ToC or Class ToC

IRI: https://w3id.org/rdip/RandomSeed

A special Parameter subclass for random seed values. Explicitly typed because stochastic reproducibility depends critically on recording the exact seed used in all random number generators (NumPy, PyTorch, Python random, CUDA). Set rdip:parameterName='random_seed' and rdip:parameterValue to the integer seed string.
has super-classes
Research parameter c

Research activityc back to ToC or Class ToC

IRI: https://w3id.org/rdip/ResearchActivity

Any discrete activity in the research lifecycle, including planning, data collection, processing, analysis, publishing and preservation. Subclass of prov:Activity so that all PROV-O properties (wasAssociatedWith, used, generated, wasInformedBy, qualifiedAssociation) are directly applicable.

follows op, has activity op, precedes op
